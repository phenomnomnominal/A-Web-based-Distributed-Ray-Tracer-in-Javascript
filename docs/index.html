index.coffee

¶

index.coffee is the entry point into the server code. The compiled version (index.js) is run by calling 

node javascript/server/index.js

from the command line. This launches the server and initialises it for accepting requests from clients.

Path-specific request handlers:
             		                            
             		           
                               
                            
                 ¶               
               
Specific routing functions from requestHandlers are associated with their respective paths in the handlers object.
             		                            
handlers = 
             		           
                               
                            
                 ¶               
               
    

  • Requests to '/' and '/master' are associated with the master function. The master function generates the render upload page, which allows the user to upload and initialise a rendering operation.
    • 

             		                            
  
  '/': requestHandlers.master
  '/master': requestHandlers.master
  
             		           
                               
                            
                 ¶               
               
    

  • Requests to '/upload' are associated with the upload function. When the user uploads a COLLADA file, it is first parsed by sceneHandler.coffee, then the resulting render object is inserted into the database by database.coffee and finally a shortened URL (using the format from the Google URL Shortener) is generated by urlShorten.coffee which is returned to the user. The shortened URLs generated by upload use the UUID Version 4 to identify each render:
    • 

             		                            
  
  '/upload': requestHandlers.upload
             		           
                               
                            
                 ¶               
               


render?renderId=XXXXXXXX-XXXX-4XXX-YXXX-XXXXXXXXXXXX



             		                            
  
             		           
                               
                            
                 ¶               
               
    

  • Requests to '/render' are associated with the render function. The render function generates the rendering page, which first posts the renderID from the URL to '/getRender', which returns the appropriate render object from the database.
    • 

             		                            
  
  '/render': requestHandlers.render
  
             		           
                               
                            
                 ¶               
               
    

  • Requests to '/getRender' are associated with the getRender function. When a user requests a shortened URL provided by the upload function, the corresponding renderID is posted to *'/ This retrieves the render object from the database and sends it back to the user via a WebSocket.
    • 

             		                            
  
  '/getRender': requestHandlers.getRender
             		           
                               
                            
                 ¶               
               
    

  • Requests to '/test' are associated with the test function. The test function initialises and perform all of the QUnit regression and unit tests.
    • 

             		                            
  
  '/test': requestHandlers.test
